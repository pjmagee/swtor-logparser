<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local disk `settings.json` → overlay | The settings file is a local, app-written file but is treated as untrusted on read (a user or other process may corrupt/edit it). Parse must be robust. |
| overlay → local disk `settings.json` | The overlay writes window geometry + font size. Must write no secrets and to a fixed, non-user-controlled path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Denial of Service | Corrupt/malformed settings.json crashes the overlay on startup | mitigate | D-06: `SettingsService.Load()` wraps read+deserialize in try/catch and returns defaults on ANY failure; verified by the corrupt-file unit test + human-check step 3. |
| T-09-05 | Tampering / Path traversal | Persisted values used to build a file path | mitigate | The settings path is a fixed `%LocalAppData%\SwtorLogParser\settings.json` computed from `SpecialFolder.LocalApplicationData`; persisted fields are numeric window geometry + font size only — never used as a path. |
| T-09-06 | Information Disclosure | Secrets written to the settings file | accept | Only window geometry + font size are persisted; no credentials/tokens/PII exist in this app to leak. Nothing sensitive is written. |
| T-09-07 | Elevation of Privilege | Writing under %LocalAppData% | accept | Per-user, non-elevated location; no system-wide write, no privileged operation. |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan — settings use the in-box `System.Text.Json`; no new packages added. |
</threat_model>
