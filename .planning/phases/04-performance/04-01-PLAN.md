<objective>
PERF-01: Eliminate the per-line `char[]` allocation and the count-via-full-parse in `CombatLog.cs`. Introduce ONE private offset-tracking line splitter over the `ReadToEnd()` backing string that is the single source of truth for BOTH `ToString()`'s line count and `GetLogLines()`'s zero-copy slices. Each `CombatLogLine` references a window (`string.AsMemory(start, length)`) into the single heap string returned by `ReadToEnd()` — true zero-copy, preserving the `ReadOnlyMemory<char>` intent end-to-end. `ToString()` counts lines WITHOUT building `CombatLogLine` objects.

This is a PURE perf refactor — output of `GetLogLines()` must be IDENTICAL. Wave-0 tests are written FIRST (test-first within this plan) to lock the read/slice behavior and the splitter's parity with `MemoryExtensions.EnumerateLines` before the optimization lands.

Purpose: Restore the project's core zero-allocation span/memory design where `.ToArray()` broke it (PERF-01).
Output: Refactored `CombatLog.cs` (no per-line `char[]`, parse-free count) + new `CombatLogReadTests.cs` Wave-0 tests.
</objective>
