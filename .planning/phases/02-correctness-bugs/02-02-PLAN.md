<objective>
Make the GameObject / Ability / Action numeric parsing graceful (BUG-05, the EAGER + inherited-lazy
subset) AND make the two shared static parse caches thread-safe (BUG-06), in one plan because these
changes are coupled: `GameObject.Parse` reads `.Id` eagerly, so the BUG-05 TryParse change and the
BUG-06 cache rewrite touch the SAME `GameObject.Parse` method and must land coherently.

Purpose: A non-numeric brace id must skip the line (null) instead of crashing the reader task
(BUG-05); and the background reader writing the cache concurrently with a host parse must not corrupt
the dictionary or throw `InvalidOperationException` (BUG-06). Both are correctness defects with no
external-contract change (`static T? Parse(...)` stays; the cache KEY stays `Rom.GetHashCode()` —
the key redesign is RFCT-03/Phase 3).

Output: `CombatLogs.cs` caches → `ConcurrentDictionary`; `GameObject.cs` GetId/GetParentId → `ulong.TryParse`
and a conditional (NOT blind `GetOrAdd`) cache write; `Ability.cs`/`Action.cs` cache writes → `TryAdd`.
Two flipped characterization tests (GameObject eager, Ability lazy) plus retained goldens and the
graceful-null Action matrix. Suite green, zero skips.
</objective>
