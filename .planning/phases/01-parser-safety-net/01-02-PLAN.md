<objective>
Characterize the three LAZY-throwing parse factories — `Actor`, `Threat`, `Value`. Per the verified eager/lazy map (RESEARCH.md), these `Parse` methods construct successfully on malformed input and defer numeric parsing to property getters: `Actor.Health/MaxHealth/Id` (Actor.cs:64/73/93/100/107), `Threat.Value` (Threat.cs:14), `Value.Id` (Value.cs:47). A naive `Assert.Null(Threat.Parse("<abc>"))` would FAIL because `Parse` returns a non-null object. This plan locks current behavior by asserting the throw ON THE PROPERTY (`Assert.Throws`), adds guard-null `[Theory]` matrices where the guards are already graceful, delimiter-in-name `[Theory]` matrices, a position-locale golden, and golden-line locks (TEST-03).

Purpose: These three models are where Phase 2's BUG-05 (TryParse) and BUG-03 (InvariantCulture) fixes will land. Each `Assert.Throws` characterization here is the exact test Phase 2 will invert to `Assert.Null` / a graceful value once the production fix ships.
Output: Extended `ActorTests.cs`, `ThreatTests.cs`, `ValueTests.cs` — new tests only, no production change.
</objective>
