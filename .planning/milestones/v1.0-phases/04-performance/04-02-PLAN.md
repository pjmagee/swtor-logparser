<objective>
PERF-02: Replace the `Console.Clear()` + full redraw per event in the Native CLI `monitor` render with in-place cursor repositioning. Use `Console.SetCursorPosition(0, row)` + per-row pad-to-width overwrite (copying the in-repo `OnCombatLogAdded` analog that already does `SetCursorPosition` + space-pad), guard `Console.IsOutputRedirected`, clamp width/height, and clear rows that were drawn last frame but not this frame (row count shrank). This eliminates the full-screen repaint flicker.

Scope is the Native AOT CLI host ONLY. The visible text format is UNCHANGED — only the repaint mechanism changes. The `list` command is untouched. The managed CLI (`SwtorLogParser.Cli`, System.CommandLine.Rendering) is OUT OF SCOPE (replaced in Phase 5).

NOTE FOR THE VERIFIER: There is NO automated console test — console rendering is not unit-testable. PERF-02 is verified by code review + a manual flicker check (run `dotnet run --project SwtorLogParser.Native.Cli -- monitor`). This is called out per 04-VALIDATION.md so the verifier does not expect an automated flicker test. This plan is `autonomous: false` because it ends in a human-verify checkpoint.

Purpose: Remove the full-screen flicker on every stats event (PERF-02).

    Refactor `Update(SlidingExpirationList list, CombatLogsMonitor.PlayerStats playerStats)` in SwtorLogParser.Native.Cli/Program.cs per PERF-02 (locked decisions in 04-CONTEXT.md):

    1. Keep `list.AddOrUpdate(playerStats);` as the first line.
    2. REMOVE the `Console.Clear();` call entirely (this is the flicker source — DO-NOT-BREAK / the whole point of PERF-02).
    3. Add a `private static int _lastRowCount;` host-local render-state field (per 04-RESEARCH Pattern 3).
    4. Redirected-output guard: if `Console.IsOutputRedirected`, fall back to plain `Console.WriteLine` per row (NO cursor calls — `SetCursorPosition`/`WindowWidth` throw or misbehave when redirected, per Pitfall 5) and return without touching `_lastRowCount`/cursor.
    5. Interactive path: compute `int width = Math.Max(1, Console.WindowWidth - 1);` (clamp so a 0/odd width cannot break PadRight). For each `item` in `list.Items`, `Console.SetCursorPosition(0, 1 + row)` (row 0 stays the filename header from OnCombatLogAdded — DO NOT start at row 0), build the text with the IDENTICAL format string and arguments as today:

Write it padded/truncated to `width` (e.g. `text.Length > width ? text[..width] : text.PadRight(width)`) so a previously longer row is fully overwritten. Increment `row`.
    6. After the loop, clear any rows that existed last frame but not this frame: `for (int r = row; r < _lastRowCount; r++) { Console.SetCursorPosition(0, 1 + r); Console.Write(new string(' ', width)); }`. Then set `_lastRowCount = row;`.
    7. (Discretion) Optionally clamp the maximum in-place row to `Console.WindowHeight - 1` to avoid `SetCursorPosition` exceeding the buffer height; with the 30s SlidingExpirationList this is an edge case — only add if cheap.

    Do NOT touch `OnCombatLogAdded` (keep its row-0 header behavior), `ListCombatLogs`, `MonitorCombatLogs` wiring, or the format/`"N"`/`"-"` rendering. Keep the text byte-for-byte identical to today.
