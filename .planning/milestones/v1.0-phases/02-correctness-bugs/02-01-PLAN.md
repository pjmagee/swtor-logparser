<objective>
Harden the LAZY numeric parse getters in three model types — `Threat`, `Actor`, `Value` — so that
malformed (non-numeric) field content returns `null` from the property getter instead of throwing
`FormatException` during live log tailing (BUG-05, the leaf-parse subset with NO cache coupling).
Each production edit lands in the SAME commit as the inversion of its Phase-1 characterization
test (`Assert.Throws` → graceful `Assert.Null`), keeping `dotnet test` green every commit.

Purpose: Malformed log lines must not crash the background reader task. This is the V5
(Input Validation) remediation for the three model types that have no shared-cache dependency,
so they can be fixed independently of the GameObject/Ability/Action cache work (plan 02).

Output: `Threat.cs`, `Actor.cs`, `Value.cs` converted to `TryParse`; the three flipped
characterization tests plus retained positive goldens. Suite green, zero skips.
</objective>
