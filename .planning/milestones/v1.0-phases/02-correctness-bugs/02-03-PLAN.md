<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SWTOR combat-log file → CombatLogLine timestamp parse | A locale-variant or non-parseable first section previously threw `FormatException` from `Parse` and crashed the reader. |
| SWTOR settings dir → CombatLogs static ctor | A `*PlayerGUIState.ini` filename without '_' threw `IndexOutOfRange` → `TypeInitializationException` at startup. |
| Monitor lifecycle (Start/Stop) → background worker tasks | Stop() failed to cancel the workers (wrong token) and threw NRE if called before Start. |
| Parser ↔ live game writer (open file handle) | The log is opened while the running game actively appends to it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering / DoS | `CombatLogLine.Parse` timestamp | mitigate | `DateTime.TryParseExact` + `InvariantCulture` + explicit `HH:mm:ss[.fff]` formats in the static factory; bad timestamp → null (skip), locale-stable (BUG-03) |
| T-02-07 | Denial of Service | `CombatLogs` static ctor | mitigate | `SecondSegmentOrNull` length guard skips filenames without '_' (BUG-04); no startup crash |
| T-02-08 | Resource exhaustion / availability | `CombatLogsMonitor` Start/Stop | mitigate | Pass the linked CTS token to workers (BUG-01) so Stop() cancels them; null-guard Stop() (BUG-02) so a pre-Start Stop is a safe no-op |
| T-02-09 | Tampering (least-privilege) — MUST NOT block the live writer | `CombatLog.GetLogLines` file open | mitigate | `FileAccess.Read` (least privilege) but `FileShare.ReadWrite` PRESERVED so the live SWTOR client writer is never blocked (BUG-07, planning constraint) |
</threat_model>
