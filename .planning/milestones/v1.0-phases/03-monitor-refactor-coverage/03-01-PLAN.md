<objective>
Implement RFCT-02 (make `CombatLogsMonitor` constructible in every build configuration via a public DI constructor, with `Instance` always defined) and the TEST-01 monitor-lifecycle/Rx-delivery coverage that the new public ctor + internal test seam make possible.

Purpose: Today `Instance` is only defined for configs literally named `RELEASE` or `DEBUG` (the `#if RELEASE/#elif DEBUG` block has no `#else`), so any other config leaves `Instance` undefined and the core lib fails to compile; both constructors are `private` so the monitor cannot be constructed for tests. This plan removes that gap and adds the lifecycle coverage deferred from Phase 2 (BUG-01).
Output: An always-defined `NullLogger`-backed `Instance`, a public `ILogger<CombatLogsMonitor>` ctor, an internal push seam, and `CombatLogsMonitorTests` proving construction + Start/Stop/Rx delivery.
</objective>
