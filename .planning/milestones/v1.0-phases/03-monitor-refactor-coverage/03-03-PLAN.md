<objective>
Implement RFCT-01: collapse the byte-identical, UI-free `View/Entry.cs` + `View/SlidingExpirationList.cs` from CLI and Native.Cli into ONE shared core-library copy in a new `SwtorLogParser.View` namespace; point both CLI hosts at the core copy and DELETE their duplicates; and refactor the Overlay's WinForms `SlidingExpirationList` (a `BindingList<Entry>`/`DataGridView` adapter) to WRAP/compose the shared core sliding-expiration logic so the expiration logic lives in exactly ONE place ŌĆö while keeping ALL WinForms types host-side (NONE enter the AOT core lib).

Purpose: The `Entry`/`SlidingExpirationList` types are triplicated. CLI and Native.Cli variants are byte-identical and UI-free, so they promote verbatim. The Overlay variant is a true WinForms adapter and must stay host-side, but per success criterion 1 the *expiration logic* must live in exactly one place ŌĆö so the Overlay adapter composes the core list rather than re-implementing the expiry loop.
Output: A shared core `Entry` + `SlidingExpirationList`, deleted CLI/Native duplicates, updated host `using`s, an Overlay adapter that composes the core expiry logic, and a test for the shared list.

Sequencing note: depends_on 03-01/03-02 expresses the research-recommended RFCT-02 ŌåÆ RFCT-03 ŌåÆ RFCT-01 ordering for the sequential executor; this plan touches no monitor/cache source files so there is no hard file conflict, but it runs after the wave-1 refactors.
</objective>
