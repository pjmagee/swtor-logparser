<objective>
Implement the filesystem-hermeticity seam (supports TEST-01/TEST-02 CI-readiness): abstract `CombatLogs`'s filesystem access — the `Environment.GetFolderPath` log/settings paths, `EnumerateCombatLogs`, and the `PlayerNames` static-ctor enumeration that throws `TypeInitializationException` on CI — behind an injectable seam guarded by `Directory.Exists`, keeping the static `CombatLogs` facade for hosts. Then make the two deferred Phase 1 non-hermetic tests (`All_Logs_Are_Not_Null`, `Player_Is_Local_Is_True`) hermetic and CI-safe.

Purpose: `CombatLogs`'s static ctor enumerates a real settings directory at type-load; on a machine without the SWTOR folders this throws `DirectoryNotFoundException` → `TypeInitializationException` that poisons the type for the whole test run. This is exactly why `All_Logs_Are_Not_Null` is RED on this machine today (76/77 at planning time) and why those two tests were deferred. The seam makes them deterministic and unblocks Phase 6 CI.
Output: An `ICombatLogSource` seam, a `Directory.Exists`-guarded default impl + test-injectable provider behind the static facade, and two hermetic tests.

Sequencing note: depends_on 03-02 because both plans edit `CombatLogs.cs` (Plan 02 adds the cache fields there) — this plan runs after Plan 02 to avoid a same-file conflict and must preserve those cache fields.
</objective>
